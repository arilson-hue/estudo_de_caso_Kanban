## Entrevista – Perguntas e Respostas

Pergunta 1: Qual é o principal problema enfrentado atualmente?

Resposta:
Atualmente temos dificuldade em acompanhar o andamento das tarefas da equipe. As atividades são controladas de forma manual, o que gera falta de visibilidade, atrasos e retrabalho.

Pergunta 2: Qual seria o principal objetivo do novo sistema?

Resposta:
O principal objetivo é ter um sistema simples que permita visualizar as tarefas em andamento, saber quem está responsável e identificar rapidamente o que já foi concluído ou ainda precisa ser feito.

Pergunta 3: Quais funcionalidades são consideradas essenciais?

Resposta:

Criar novas tarefas

Visualizar tarefas em um quadro Kanban

Alterar o status das tarefas (A Fazer, Em Progresso e Concluído)

Excluir tarefas quando necessário

 Pergunta 4: O sistema precisa de controle de usuários ou login?

Resposta:
Neste primeiro momento, não. O foco é validar o fluxo de trabalho. Funcionalidades como login podem ser consideradas em versões futuras.

 Pergunta 5: Como o sistema será utilizado no dia a dia?

Resposta:
O sistema será acessado via navegador pelos colaboradores e gestores, funcionando como um painel visual para acompanhamento das atividades da equipe.

Pergunta 6: Existe alguma prioridade para as tarefas?

Resposta:
Sim. As tarefas mais críticas devem ser identificadas facilmente, mesmo que inicialmente isso seja feito apenas por descrição ou organização visual.

Pergunta 7: Há alguma restrição técnica?

Resposta:
O sistema deve ser simples, de fácil manutenção e acessível via navegador, sem necessidade de instalação complexa.

## Requisitos Funcionais (RF)

RF01: O sistema deve permitir a criação de tarefas.

RF02: O sistema deve exibir as tarefas em um quadro Kanban.

RF03: O sistema deve permitir a alteração do status das tarefas.

RF04: O sistema deve permitir a exclusão de tarefas.

RF05: O sistema deve funcionar em navegadores web modernos.

## Requisitos Não Funcionais (RNF)

RNF01: O sistema deve possuir interface simples e intuitiva.

RNF02: O código deve ser organizado e documentado.

RNF03: O sistema deve permitir fácil manutenção.

RNF04: O projeto deve possuir testes automatizados.

RNF05: O versionamento deve ser feito utilizando GitHub.
